* Overview

This document captures the build process, methodology, issues encountered,
and sequencing decisions made during the development of cprr.

* Build Methodology

** Incremental Development

The project was built incrementally with progressive commits:

1. *Foundation* - Module initialization, zero dependencies
2. *Core CLI* - State machine with guards
3. *Build System* - Makefile with automatic variables
4. *Documentation* - README, CONTRIBUTING in org-mode
5. *CI/CD* - GitHub Actions for lint and release
6. *Tooling* - Release scripts, dependency checks

** Design Principles

| Principle | Implementation |
|-----------+----------------|
| Zero dependencies | Pure Go stdlib |
| Unix philosophy | stdin/stdout/stderr, exit codes |
| Composability | Makefile fallthrough to scripts |
| Literate programming | Org-mode documentation |
| Progressive enhancement | Guards enforce rigor |

* Issues Encountered

** Flag Parsing Order

*Problem*: Global =-h= flag intercepted command-specific help.

*Solution*: Parse global flags only before command token:
#+BEGIN_SRC go
// Only parse global flags BEFORE the command
for i := 0; i < len(args); i++ {
    if !strings.HasPrefix(arg, "-") {
        remaining = append(remaining, args[i:]...)
        break
    }
    // handle global flags
}
#+END_SRC

** Hypothesis Flag Conflict

*Problem*: =-h= commonly means "hypothesis" but conflicts with help.

*Solution*: Use =--hypothesis= for the long form, document clearly:
#+BEGIN_SRC
Note: The hypothesis flag uses --hypothesis (not -h) to avoid
conflict with the help flag.
#+END_SRC

** Makefile Redundancy

*Problem*: Explicit =dist/:= rule duplicated =%/:= pattern.

*Solution*: Remove redundant rule, rely on pattern:
#+BEGIN_SRC makefile
# Directory creation pattern
%/:
	install -d $@

# This uses the pattern above via | dist/
dist/$(BINARY)-%: $(GO_FILES) | dist/
#+END_SRC

** Version Detection

*Problem*: =go --version= returns error (flag not defined).

*Solution*: Special-case version commands in deps.sh:
#+BEGIN_SRC bash
case "$cmd" in
    go) version=$(go version 2>&1 | head -1) ;;
    tmux) version=$(tmux -V 2>&1 | head -1) ;;
    *) version=$("$cmd" --version 2>&1 | head -1) ;;
esac
#+END_SRC

* Architecture Decisions

** Single File vs Packages

*Decision*: Single =main.go= file.

*Rationale*:
- Simple CLI, ~1000 lines
- Easy to understand at a glance
- No import cycles to manage
- Can refactor later if needed

** State Machine as Data

*Decision*: Define transitions as data, not code.

#+BEGIN_SRC go
var transitions = []Transition{
    {From: StatusOpen, To: StatusTesting, Guards: []Guard{...}},
    ...
}
#+END_SRC

*Rationale*:
- Declarative, easy to verify
- Maps directly to formal specification
- Guards are composable functions

** JSON Storage

*Decision*: JSON file in =~/.cprr/= or =./.cprr/=.

*Rationale*:
- Human-readable, debuggable
- No database dependency
- Easy backup/sync
- Local override for project-specific tracking

* Lessons Learned

1. *Start with the state machine* - Core logic first, CLI second
2. *Guards as functions* - Composable, testable, formally verifiable
3. *Makefile patterns* - =%/:= and fallthrough reduce boilerplate
4. *Org-mode for docs* - Tangleable, exportable, version-controlled
5. *Progressive commits* - Each commit is a coherent unit with notes
6. *Git notes* - Capture context that doesn't fit in commit message
